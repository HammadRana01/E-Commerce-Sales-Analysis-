E-Commerce Sales Analysis (SQL Project)

This project performs an end-to-end analysis of an e-commerce dataset using SQL.  
The goal is to extract business insights such as revenue trends, customer behavior, and product performance.

The project demonstrates practical SQL skills used in real data analyst roles.

---

Project Structure

schema.sql  
Creates the database tables used in the project:
- orders
- order_details
- sales_targets

queries.sql  
Contains analytical SQL queries used to generate business insights such as:
- KPI calculations
- Sales analysis
- Customer insights
- Category performance
- Target vs actual sales comparison

Analysis_SQL.ipynb  
Jupyter Notebook used to run and document SQL analysis step-by-step.

---

Business Questions Answered

- What is the **total revenue, profit, and number of orders?**
- What is the **average order value?**
- Which **states generate the highest revenue and profit?**
- Which **product categories perform best and worst?**
- What are the **top selling sub-categories?**
- How do **sales and profit change monthly?**
- Are **sales targets being achieved?**
- Who are the **top customers by spending?**

---

Key Insights

- Identified top performing states based on total revenue.
- Found the highest and lowest performing product categories.
- Analyzed monthly trends in sales and profit.
- Compared actual sales performance against target sales.
- Discovered the top customers contributing to revenue.
